# Glossary

Alphabetical definitions for terms used throughout the codebase and wiki.

---

**agent_state**
The lifecycle state of a Claude (or other agent) pane as reported by hook shims. Values: `""` (no hook event yet / unknown), `"working"` (turn in progress), `"idle"` (turn complete, waiting for input), `"attention"` (permission request or blocking prompt). Drives the stoplight for Claude panes. See [concepts/stoplight.md](../concepts/stoplight.md) and [concepts/hook-shims.md](../concepts/hook-shims.md).

---

**DAEMON_TOKEN**
A random bearer token generated by `install-station.sh` and stored at `/etc/reck-stationd/token` (mode 0600). The daemon reads it at startup and publishes it into `DAEMON_TOKEN` in its own environment. All HTTP and WebSocket requests from the satellite must carry it as `Authorization: Bearer <token>`. Loopback requests to `/panes/:id/agent-event` are exempted so hook shims don't need the token in their child environment. See [concepts/auth.md](../concepts/auth.md).

---

**hook shim**
A small shell script (`reck-claude-hook.sh`) installed into `~/.claude/hooks/` by the daemon at startup. Claude Code invokes the shim on lifecycle events (SessionStart, UserPromptSubmit, PreToolUse, PostToolUse, Stop, etc.); the shim POSTs the event to `POST /panes/:id/agent-event` on the daemon, driving the stoplight state machine. The marker `reck-hook-v1` identifies Reck-owned entries in `~/.claude/settings.json`. See [concepts/hook-shims.md](../concepts/hook-shims.md).

---

**Hybrid mode**
The default. Both the local daemon and a configured station daemon are enabled at the same time; each project lives on exactly one host. See [concepts/modes.md](../concepts/modes.md).

---

**Local**
The satellite spawns `reck-stationd` as a child process bound to `127.0.0.1:7315`. No bearer token is required. Quitting the satellite stops the local daemon after a confirmation dialog (see "quit stops the local daemon" in [concepts/behaviors.md](../concepts/behaviors.md)). Pairs with Station in hybrid mode, or runs alone for single-Mac users. See [concepts/modes.md](../concepts/modes.md).

---

**loopback exemption**
Hook shims POST lifecycle events to the daemon from inside a pane's child process. To avoid leaking `DAEMON_TOKEN` into every pane's environment, the daemon's `authMiddleware` skips token verification for requests to `/panes/:id/agent-event` that arrive from a loopback address (`127.0.0.1`, `::1`, or IPv4-mapped `::ffff:127.0.0.1`). Remote (tailnet) callers to that path still require the token. Source: `daemon/internal/http/router.go:isLoopbackAddr`, `isAgentEventPath`.

---

**managed root**
The directory `ManagedProjectsRoot` (`/Users/reck-connect/projects` by default, declared as a `var` in `daemon/internal/config/config.go` so tests can override it). When `AddProjectRequest.cwd` is omitted, the daemon creates the project directory here. Projects under the managed root may be deleted on `DELETE /projects/:id`; projects outside it are left on disk.

---

**pane**
A PTY-backed process managed by the daemon. Kinds: `claude` (Claude Code), `shell` (login shell), `codex` (Codex CLI). Each pane has a unique `pane_id` (`p_<12 hex chars>`), belongs to exactly one project, and exposes a WebSocket endpoint for terminal I/O. Panes are not persistent across daemon restarts; sessions (for Claude) and slots (for shell) provide resume continuity. See [concepts/panes.md](../concepts/panes.md).

---

**preamble**
A per-project string injected as `--append-system-prompt` when spawning Claude panes. Configured in `projects.toml` via the `preamble` field, or via `AddProjectRequest.preamble`. A daemon-level baseline preamble is also spliced in (controlled by `RECK_SATELLITE_HINT` and `RECK_DISABLE_BASELINE_PREAMBLE` env vars). See [concepts/preamble.md](../concepts/preamble.md).

---

**PTY**
Pseudo-terminal. The daemon creates a PTY pair for each pane (`daemon/internal/pty/`) and runs the pane process (`claude`, `zsh`, etc.) inside it. The PTY's output is streamed to connected WebSocket clients as base64-encoded bytes; input from the satellite is written back to the PTY's master end.

---

**project**
A registered development directory. Projects live in `projects.toml` (or are created via the API). Each has an `id` (URL-safe slug, 1–64 chars), `name`, `cwd`, and optional fields (`default_pane`, `shell`, `preamble`, `display_name`). The satellite's left rail shows one card per project. See [concepts/projects.md](../concepts/projects.md).

---

**restore candidate**
A session or shell slot the daemon believed was live on the previous run but whose pane is no longer running. Surfaced via `GET /restore-candidates`. The satellite shows a prompt at reconnect; the user can restore (re-spawn the pane with the saved identity) or skip. See [concepts/sessions.md](../concepts/sessions.md).

---

**satellite**
The Electron desktop app (`satellite/`) that runs on the laptop. Renders panes via xterm.js, manages project registration, and proxies IPC between the renderer and the daemon (or the spawned daemon child in Local mode). Config stored in `~/Library/Application Support/Reck Connect Satellite/config/settings.json`.

---

**session_id**
A stable RFC 4122 v4 UUID assigned to each Claude pane at spawn. Persisted in the sessions store. Used to resume transcripts via `claude --resume <uuid>` and to rekey saved layouts across daemon restarts. Shell panes use `slot_id` instead; `session_id` is empty for non-Claude panes. See [concepts/sessions.md](../concepts/sessions.md).

---

**slot_id**
A stable RFC 4122 v4 UUID for shell panes (the shell analogue of `session_id`). Restore re-spawns the shell with the same argv captured at the original create. See [concepts/sessions.md](../concepts/sessions.md).

---

**station**
The always-on Mac that runs `reck-stationd`. Projects live here; Claude Code panes run here. The satellite connects to the station over Tailscale.

---

**Station**
The satellite connects to an existing `reck-stationd` on a remote station Mac over Tailscale. Requires a `stationUrl` and a `daemonToken`. All requests carry `Authorization: Bearer <token>`; WebSocket upgrades carry the token as a `Sec-WebSocket-Protocol` subprotocol. Pairs with Local in hybrid mode. See [concepts/modes.md](../concepts/modes.md).

---

**stoplight**
A per-pane status indicator with four values: `gray` (no output yet / unknown), `orange` (active / working), `green` (idle / done), `red` (error or needs attention). For Claude panes, driven by hook-shim `agent_state`; for shell panes, by PTY byte-flow heuristics. The project-level stoplight is the max severity across all panes. See [concepts/stoplight.md](../concepts/stoplight.md).
